# ADR-0006: マルチテナント分離を最初から徹底する

- ステータス: 採用
- 日付: 2026-05-31

## コンテキスト
本部・チェーン・複数店舗を扱うため、テナント間のデータ混在は致命的。

## 決定
- すべての業務テーブルに `tenant_id`。読み書きは `TenantId` でスコープする。
- EF Core の Global Query Filter（または必須引数リポジトリ）でテナントスコープを機械的に強制。
- API は認証コンテキストからテナントを解決し、テナント外リソースは 403/404。
- 権限は本部管理者 / 店舗管理者 / 閲覧者を分離。

## 影響
- すべてのクエリ・テストでテナント前提を考慮する必要がある。
- テナント分離はテストの必須観点とする。
