# ADR-0004: Human-in-the-loop（承認なしに配信しない）

- ステータス: 採用
- 日付: 2026-05-31

## コンテキスト
AI による自動配信は誤配信・説明責任・現場不信のリスクが大きい。企画方針として「AI提案 → 管理者確認 → 承認 → 配信」を採る。

## 決定
- `MatchingCampaign` の状態遷移で `proposed → approved` を人手のみとする。
- `approved` を経ずに `sent` へ遷移できない実装にする（Domain で不変条件、API で 409/422）。
- 監査ログに提案・承認・編集・配信を記録する。

## 影響
- 完全自動化はしない（将来も既定は承認フロー）。
- 現場の納得とリスク低減を優先。承認 UI（編集可能）を前提に設計する。
