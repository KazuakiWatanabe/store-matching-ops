# ADR-0003: スコアリングはルールベース（v0）から始める

- ステータス: 採用
- 日付: 2026-05-31

## コンテキスト
来店可能性スコアは現場の納得と改善容易性が重要。初期はデータが薄く、機械学習は説明性・コールドスタートで不利。

## 決定
- Phase 0/1 は機械学習を使わず、ルール + 重み付けスコアを採用する。
- v0 は 3 要素（休眠日数 / 来店周期との乖離 / 空き枠の曜日・時間帯一致）。重みは設定で注入。
- 欠損項目は重みから除外し、残り項目で動的に再正規化する（段階的スコアリング）。
- 機械学習の導入は別 ADR を起票してから（データ蓄積後に検討）。

## 影響
- 提案理由が常に説明可能（`ScoreBreakdown`）。
- 重みの調整が容易（設定変更）。
- 精度の上限はあるが、増分実証（ADR-0007）で効果を確認しながら拡張する。
